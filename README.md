# 🚀 Dominando o Bubble.io com NotebookLM

Este repositório contém meu projeto prático para o desafio da **DIO**, focado em utilizar o **NotebookLM** como uma ferramenta de aprendizagem ativa para dominar a plataforma No-Code **Bubble.io**.

---

## 🎯 Contexto e Objetivos
O objetivo deste caderno temático é consolidar conceitos fundamentais e avançados do Bubble.io, focando em:
- Estruturação de Banco de Dados Relacional.
- Lógica de Workflows e Estados Personalizados (Custom States).
- Otimização de performance e segurança (Privacy Rules).

## 🧠 Engenharia de Prompts e "Cicatrizes"

Durante o uso do NotebookLM, testei diferentes abordagens para extrair o melhor da IA:

1.  **Prompt Inicial:** "Como funciona o banco de dados do Bubble?"
    * *Resultado:* Muito genérico.
2.  **Prompt Refinado:** "Explique a diferença técnica entre 'Data Type' e 'Option Set' no Bubble, focando em quando usar cada um para economizar WUs (Workload Units)."
    * *Resultado:* Resposta precisa sobre performance e carregamento do lado do cliente vs. servidor.
3.  **Dificuldade Encontrada:** A IA às vezes sugeria plugins antigos. Precisei adicionar o comando: *"Considere apenas as funcionalidades nativas da versão mais recente do motor de responsividade."*

## 📖 Miniguia de Estudo (Entrega Final)

### 1. Glossário de Conceitos Chave
* **Custom State:** Variável temporária que armazena dados na página sem precisar consultar o banco de dados (rápido e gratuito).
* **Privacy Rules:** Regras de segurança no servidor que protegem os dados de usuários não autorizados.
* **API Connector:** Ferramenta para conectar seu app Bubble a qualquer outro serviço (Google, OpenAI, Stripe).

### 2. Prompts Reutilizáveis para Revisão
* > "Analise minha estrutura de dados para um Marketplace e sugira as Privacy Rules ideais."
* > "Crie um passo a passo para configurar um Workflow de 'Esqueci minha senha' seguindo as fontes fornecidas."
